# 送给学弟的前端学习规划

# 前言

前端在网络上算是鄙视链偏底端的了，反正前端只是写页面的，后端不过增删改查，运营就是画大饼，总统每天坐飞机。

其实，前端的本质是人机交互软件开发。如果在你心中，前端的定位仅仅是做网页的，那么前端必定是没有未来的。

在我看来前端没有特别难以理解的技术，长期学习过程中，前端带来的难点可能更多在于要学习不断更新的技术方案和业务方案。但不论是学习了jQuery还是Vue、React，我们不能离开职业的本质——交互体验，这样你才会把握好学习的方向。

如果是觉得前端简单才学前端我也建议你多考虑下，现在大厂一个个都说自己急缺前端，没见人才市场上这么多“前端”出身的工作都找不到。

# 学习建议

# 1 前端基础

  1. 真基础

    • HTML（html权威指南）
    • CSS（css权威指南，css揭秘）
    • JavaScript（红宝书、犀牛书——枕头书，多读）
    • Web API

  2. 容易被忽视却极为重要的知识

    • 浏览器（运行机制、渲染原理）（虽然只有一条，架不住多）

  3. 还是基础

    • ES6+（读阮一峰的或者《深入理解ES6》）

    • 《http权威指南》《JavaScript DOM编程艺术》

    • 《JavaScript数据结构与算法》（没读过，感觉学了数据结构就够了嘛，不过推荐的人挺多的）

  4. 这些读完了JavaScript应该不错了，可以大胆尝试看一些源码

    • 《你不知道的JavaScript》
    • 《JavaScript语言精粹》
    • 《JavaScript函数式编程》（正在读感觉不错，不仅可以学习函数式编程，也能复习ES6）
    • 设计模式

# 2 计算机基础

  1. 全忘完了，现在感觉血亏
  • 计算机网络
  • 数据结构、算法（建议现在开始坚持力扣上刷题）
  • 操作系统
  • 安全（了解一点web安全即可，CSRF，XSS）
  • 编译原理（没这课，自学）

# 3工程化

  1. 构建工具

    • webpack（看完这个就不错了，vite这种新兴的和Glup这些老玩意有精力可以看看）
    • npm script （了解）

  2. 版本管理工具

    • Git（学点基础就行了，想深入我觉得很难）
    • svn（了解基本概念）

  3. Node相关

    • npm、yarn、npx、nvm（虽然多，都是些简单的工具而已）
    • Node.js（走前端的童鞋来一起学）

  4. css预处理器（了解就行，用的时候再学）

    • Sass
    • Less
    • Stylus

  5. 两大框架（ng感觉药量）

    • React （网上有关React和Vue谁更好的问题，到处撕，这样没意思，都学就是了）
    • Vue （全家桶包含的其他东西，自己学的时候自然就了解了，这里不一一列出）

  6. 测试（不会，也没真正做过测试）

# 4性能和优化和代码质量

  1. 《JavaScript性能优化》（读过一点，刚刚又翻了下目录，感觉可以读）
  2. 《高性能JavaScript》（没读过、还是推荐的人太多了）
  3. TS（看官网或者随便找个视频看完就行）
  4. 规范
    • ESlint
    • prettier
  5. Airbnb JavaScript风格指南（太懒了，没看完）
  6. 错误监控
    • 资源加载错误
    • 代码运行错误
  7. webpack优化（学习中）
  8. SEO优化
  9. 其他……

# 5. 写代码

多写代码，“空谈误国、实干兴邦”。 尝试写vscode插件，npm提交点你的东西，搭建博客（重要的是持续地往博客里添加内容）

# 6.新技术

  1. ES每年更新，每年都要看
  2. 什么微前端、大前端，得了解吧
  3. PWA
  4. Serverless

前端的东西在我现在看来还是太多太杂了，很多东西都没精力学，比如移动端、跨端、可视化、智能化……

但学这些只是基础，重要的还是跟别人合作开发，学会利用这些技术产生价值。

基础部分挨着学就是，其他的怎么学你学到哪儿的时候自然就知道了，什么学习路线的网上很多，还是一步一步来最好。我就不谈自己的学习方式了，弯路走的太多。

前端工程师归根结底是软件工程师，所以后端、服务器、数据库，也要学，不说多了解，基本的还是要知道的。

上面指出的基础一定要好好看，其他的不敢说对，基础多看看是没坏处的。

以上列出的都可以来找我交流，一起进步!
